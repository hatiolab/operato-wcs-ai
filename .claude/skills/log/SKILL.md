# 작업 로그 기록

현재 세션에서 수행한 작업 내용을 `.ai/logs/` 디렉토리에 **오늘 날짜 파일**(YYYY-MM-DD.md)로 기록합니다.

## 실행 절차

1. 오늘 날짜를 `YYYY-MM-DD` 형식으로 구한다.
2. `.ai/logs/` 디렉토리가 없으면 생성한다.
3. `.ai/logs/YYYY-MM-DD.md` 파일이 이미 존재하면 **기존 내용을 유지하고 하단에 추가**한다.
4. 파일이 없으면 새로 생성한다.

## 기록 형식

파일이 새로 생성되는 경우 아래 헤더로 시작한다:

```
# 작업 로그 — YYYY-MM-DD
```

각 로그 항목은 아래 형식으로 추가한다:

```
## HH:MM — 작업 제목

### 수행 내용
- 구체적인 작업 내용을 bullet으로 기술

### 변경 파일
- `path/to/file.ext` — 변경 사유 한 줄 요약

### 비고
- 특이사항, 후속 작업, 주의사항 등 (없으면 생략)
```

## 작업 내용 수집 방법

- 현재 대화에서 수행한 **모든 작업**을 시간순으로 정리한다.
- 사용자가 `$ARGUMENTS`로 요약 메시지를 전달하면 해당 내용을 중심으로 기록한다.
- 사용자가 메시지를 전달하지 않으면 대화 컨텍스트에서 수행한 작업을 자동으로 요약한다.
- 파일 생성/수정/삭제 이력이 있으면 반드시 **변경 파일** 섹션에 포함한다.
- 시간은 현재 시각을 기준으로 한다.

## 주의사항

- 기존 파일이 있으면 절대 덮어쓰지 않고 **하단에 append** 한다.
- 로그는 한국어로 작성한다.
- 불필요하게 장황하지 않게, 핵심만 간결하게 기록한다.
