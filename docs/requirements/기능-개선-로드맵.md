## 1. 경쟁 제품 대비 부족한 기능

### 1-1. Warehouse Execution System (WES) 통합 계층

Manhattan Associates 등 선두 업체는 WCS와 WMS 사이에 **WES(Warehouse Execution System)** 계층을 두어 설비 간 작업 흐름을 통합 오케스트레이션한다. 현재 Operato WCS는 ECS 모듈별로 독립적인 주문 읽기 방식(DB 기반)이며, WES 계층 도입 시 다음을 기대할 수 있다.

**현재 구조 (AS-IS)**
```
WCS 주문 테이블 ──▶ ECS 모듈 A (자기 주문 읽기)
                 ──▶ ECS 모듈 B (자기 주문 읽기)
                 ──▶ ECS 모듈 C (자기 주문 읽기)
```

**목표 구조 (TO-BE)**
```
WCS 주문 테이블 ──▶ WES 오케스트레이션 계층
                        ├──▶ 실시간 부하 밸런싱
                        ├──▶ 설비 간 자동 전환 (DAS 과부하 → P-DAS 전환)
                        ├──▶ 장애 시 대체 경로 라우팅
                        └──▶ 엔드-투-엔드 작업 흐름 관리
```

| 항목 | 내용 |
|------|------|
| 기대 효과 | 설비 간 실시간 부하 밸런싱, 장애 시 자동 대체 경로, 복합 작업 흐름 오케스트레이션 |
| 참고 제품 | Manhattan Associates WES (Gartner 17회 연속 리더), Körber WCS |
| 난이도 | 높음 (아키텍처 변경 수반) |
| 선행 조건 | 각 ECS 모듈의 실시간 상태 API 표준화 |

### 1-2. 노동력 관리 시스템 (Labor Management System)

물류 센터 운영 비용의 **50~70%가 인건비**이며, 경쟁 제품에서는 기본 기능으로 제공하고 있으나 현재 Operato WCS에는 명시적으로 포함되어 있지 않다.

#### 필요 기능

| 기능 | 설명 | 기대 효과 |
|------|------|----------|
| 작업자 생산성 추적 | 작업자별 UPH(Units Per Hour), 정확도, 동선 분석 | 성과 기반 관리 가능 |
| 인력 수요 예측 | 주문량 기반 시간대별 필요 인력 자동 산출 | 과잉/부족 인력 배치 방지 |
| 동적 작업 배정 | 실시간 작업 부하에 따라 작업자를 설비/구역 간 재배치 | 유휴 인력 최소화 |
| 인센티브/게이미피케이션 | 작업자 동기부여를 위한 성과 기반 리워드 시스템 | 이직률 감소, 생산성 향상 |
| 교육 이력 관리 | 작업자별 숙련 설비/업무 이력, 자격 관리 | 적재적소 배치 |

#### 기대 성과 (업계 벤치마크)

- 생산성 **20~40% 향상**
- 인건비 **15~30% 절감**
- 이직률 감소 (업계 평균 49% → 목표 30% 이하)

### 1-3. 옴니채널 풀필먼트

2026년 기준 업계 표준이 되어가는 기능으로, 단일 창고에서 복수 채널의 주문을 동시에 처리하는 역량이 필요하다.

| 기능 | 설명 |
|------|------|
| 통합 재고 풀 | 단일 재고에서 B2B / B2C / D2C / Click-and-Collect 동시 처리 |
| 분산 주문 관리 (DOM) | 거리, 재고, 비용 기준 최적 출고지 결정 |
| 반품 처리 고도화 | 반품 → 재판매 / 폐기 / 수리 자동 분류 및 등급 판정 |
| 채널별 SLA 관리 | 채널별 납기 기준 차등 적용 및 우선순위 자동 조정 |

> 참고: 현재 Operato WCS는 B2B/B2C 분류를 지원하지만, 채널 통합 오케스트레이션 수준에는 미치지 못한다.

### 1-4. 에너지 관리 및 지속가능성 (ESG)

최신 WCS 제품들이 경쟁적으로 도입하는 영역이다.

| 기능 | 설명 | 기대 효과 |
|------|------|----------|
| 에너지 실시간 모니터링 | 구역별/설비별 전력 소비량 추적 | 에너지 낭비 구간 식별 |
| 자동 절전 모드 | 비작업 시간 설비 자동 절전/복귀 | 최대 45% 에너지 절감 |
| 탄소 발자국 계산 | 작업 단위당 탄소 배출량 산출 | ESG 리포팅 대응 |
| 친환경 설비 연동 | 회생 제동 시스템(AS/RS), 태양광 발전 모니터링 | 에너지 자립도 향상 |

### 1-5. 보이스/웨어러블 디바이스 연동

현재 Operato WCS는 PDA/Tablet 위주의 단말 지원이며, 차세대 디바이스 레이어 확장이 필요하다.

| 디바이스 | 활용 시나리오 | 기대 효과 |
|---------|-------------|----------|
| 음성 지시 시스템 | 양손 자유 피킹, 자연어 명령 | 미스피킹 50% 감소 |
| 스마트 글래스 (AR) | AR 기반 피킹 경로 안내, 상품 위치 표시 | 신규 작업자 숙련 시간 단축 |
| 스마트 워치/밴드 | 작업자 위치 추적, 알림 수신 | 동선 분석, 안전 관리 |
| 반지형 스캐너 | 손가락 착용 바코드 스캐너 | 피킹 속도 향상 |

> 참고: ProGlove ActivityAI — 손등 착용형 웨어러블에 AI 음성 어시스턴트 탑재

## 2. AI 시대 신규 기능

### 2-1. AI 기반 지능형 주문 분배

현재 Wave 기반 규칙 분류를 ML 모델로 고도화한다.

**현재 (규칙 기반)**
```
주문 속성 (단포/합포/품목 수) → 고정 규칙 → 설비 할당
```

**목표 (ML 기반 동적 분배)**
```
┌──────────────────────────────────┐
│         ML 분배 엔진             │
│                                  │
│  입력 피처:                       │
│  · 주문 속성 (품목 수, 체적, 무게) │
│  · 현재 설비 부하율 (실시간)       │
│  · 가용 작업자 수 및 위치          │
│  · 주문 긴급도 (SLA 마감 시간)     │
│  · 과거 처리 실적 (설비별 UPH)    │
│                                  │
│  출력:                            │
│  · 최적 설비 할당                  │
│  · 예상 처리 시간                  │
│  · 대안 경로 (설비 장애 대비)      │
└──────────────────────────────────┘
```

| 항목 | 내용 |
|------|------|
| 핵심 가치 | 기존 유사 주문 분석에서 4배 성능 차이 입증 → ML 자동 최적화로 추가 향상 |
| 기술 스택 | Python ML 모델 학습 + Spring Boot 서빙 (ONNX/PMML) 또는 별도 추론 서비스 |
| 학습 데이터 | 기존 Wave 처리 이력 (주문-설비-처리시간-정확도) |

### 2-3. 예측 분석 엔진 (Predictive Analytics)

#### 수요 예측

| 항목 | 내용 |
|------|------|
| 목적 | 일별/시간대별 주문량 예측으로 사전 인력 및 설비 준비 |
| 입력 | 과거 주문 이력, 계절성, 프로모션 일정, 외부 요인 (날씨, 이벤트) |
| 출력 | 향후 N일간 시간대별 예상 주문량, 필요 인력, 설비 가동 계획 |
| 기술 | 시계열 예측 모델 (Prophet, LightGBM 등) |

#### 설비 고장 예측 (Predictive Maintenance)

| 항목 | 내용 |
|------|------|
| 목적 | 설비 고장을 사전 감지하여 계획 정비 실시 → 다운타임 최소화 |
| 입력 | 설비 가동 시간, 오류 로그, 진동/온도 센서 데이터 (가용 시) |
| 출력 | 설비별 잔여 수명 추정, 정비 권고 일정, 이상 징후 알림 |
| 기대 효과 | 비계획 다운타임 **30~50% 감소** (업계 벤치마크) |

#### AI 동적 슬로팅 (Dynamic Slotting)

| 항목 | 내용 |
|------|------|
| 목적 | 상품 보관 위치를 출고 패턴에 따라 자동 최적화 |
| 원리 | 상품 이동 패턴 학습 → 자주 함께 피킹되는 상품을 인접 배치 |
| 기대 효과 | 이동 거리 **30~55% 감소** (업계 벤치마크) |
| 참고 | 현재 DPS 주력 상품 버퍼 보관 프로세스(R1×R2 RANK 기반)를 ML로 고도화 |

### 2-4. Agentic AI 운영 어시스턴트

LLM 기반의 에이전트형 AI가 운영자의 의사결정을 실시간으로 지원한다.

#### 핵심 시나리오

**시나리오 1 — 실시간 상황 질의**
```
운영자: "오늘 오후 주문이 급증하고 있는데 어떻게 대응해야 하지?"

AI Agent: "현재 상황 분석 결과:
  - DAS 1~3번 랙: 가동율 92% (과부하 임박)
  - P-DAS 구역: 가동율 45% (여유)
  - 작업자 A, B: 피킹 구역 대기 중

  권장 조치:
  1. B2C 합포장 주문 중 3건 이하를 P-DAS로 전환
     → 예상 30분 단축
  2. 작업자 A를 DAS 2번 랙으로 재배치
     → 예상 UPH 15% 향상

  승인하시겠습니까?"
```

**시나리오 2 — 이상 감지 자동 알림**
```
AI Agent: "[알림] DAS 2번 랙 표시기 응답 지연 감지
  - 최근 5분간 응답 시간: 평균 2.3초 (정상: 0.5초 이하)
  - 과거 유사 패턴 분석: 게이트웨이 과부하 가능성 78%

  권장 조치:
  1. 해당 랙 신규 작업 일시 중단
  2. 게이트웨이 재시작 (예상 소요: 30초)
  3. 대기 주문 DAS 1번/3번 랙으로 재분배

  자동 실행하시겠습니까?"
```

**시나리오 3 — 리포팅/분석 질의**
```
운영자: "이번 달 DAS 대비 P-DAS 생산성 비교해줘"

AI Agent: "2월 생산성 비교:
  - DAS 평균 UPH: 420건 (전월 대비 +5%)
  - P-DAS 평균 UPH: 180건 (전월 대비 +12%)
  - P-DAS 건당 비용: DAS 대비 23% 저렴
  - 권장: 주문 3건 이하 합포장은 P-DAS 전환 시 월 ₩2.4M 절감 예상"
```

#### 기술 구성

| 구성 요소 | 기술 |
|----------|------|
| LLM 엔진 | Claude API 또는 자체 호스팅 모델 |
| 컨텍스트 소스 | WCS 실시간 데이터 (설비 상태, 주문 현황, 작업자 정보) |
| 액션 실행 | WCS API를 통한 제어 명령 (운영자 승인 후) |
| 인터페이스 | 채팅 UI (Manager 화면 내장) + 모바일 알림 |

| 상품 파손 감지 | 외관 이상 자동 감지 (찌그러짐, 파손) | 입고 검수, 반품 검수 |
| 포장 적정성 검사 | 상품 대비 박스 크기 자동 판별 | 포장 스테이션 |
| 적재 최적화 | 팔레트/박스 내 적재 상태 분석 | 출고 스테이션 |

> RFID 대비 저비용으로 확장 가능하며, 기존 RFID 검수(P-DAS 합포장 검수 프로세스)를 보완하는 역할
