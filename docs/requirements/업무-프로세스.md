## 1. 주문 분류 흐름 (대상 분류)

### 1-1. LV1 대상 분류 — WMS → WCS WAVE 전달

```
WMS 재고 할당
    └─▶ 1차 대상 분류 (WMS)
            ├─▶ B2B 박스 피킹
            ├─▶ B2B 합포장 피킹  ─┐
            ├─▶ B2C 박스 피킹     ├─▶ WCS I/F WAVE LIST
            ├─▶ B2C 합포장 피킹  ─┘
            └─▶ B2C 단포장 피킹
```

> WMS의 재고 할당 기준에 따라 브랜드/재고 소유주 기준으로 1차 대상 분류가 여러 번 발생할 수 있습니다.

---

### 1-2. LV2 대상 분류 — WCS WAVE → 설비별 분배

```
WCS I/F WAVE LIST
    ├─▶ B2C 단포장 피킹 ──▶ B2C 단수 프로그램
    │
    ├─▶ B2B 합포장 피킹 ──▶ 2차 대상 분류 (WMS)
    │       ├─▶ PAS 있음 ──▶ 정형 화물 합포장 ──▶ PAS 주문 처리
    │       └─▶ PAS 없음 ──▶ 비정형 화물 합포장 ──▶ DAS 주문 처리
    │                           └─▶ B2B 합포장 ──▶ DAS 주문 처리
    │
    └─▶ B2C 합포장 피킹 ──▶ B2C 합포장 주문 처리
            ├─▶ DPS
            ├─▶ DAS
            └─▶ P-DAS
```

## 2. P-DAS 업무 프로세스

### 2-1. LV1 전체 흐름

```
주문 수신 (WMS)
    └─▶ 온라인 출고 지시 (Brand → 온라인)
            └─▶ 피킹/출고
                    └─▶ 작업 게시
```

### 2-2. P-DAS 분류 프로세스 (상세)

```
Start
 └─▶ 상품 스캔 (PDA)
       ├─▶ 로케이션 할당 있음 ──▶ PDA 화면 로케이션 표시
       │       └─▶ 작업자 해당 위치 이동 ──▶ 로케이션 스캔(분류 확정)
       │               └─▶ 분류 확정 ──▶ (주문 완성 여부 판단)
       │
       └─▶ 로케이션 할당 없음 ──▶ 빈 로케이션 스캔
               └─▶ 대기 주문/로케이션 매핑
                       └─▶ 분류 확정
                               ├─▶ 주문 미완성 ──▶ 상품 스캔 반복
                               └─▶ 주문 완성 ──▶ 바구니/로케이션 매핑 ──▶ 검수/포장
```

### 2-3. P-DAS 합포장 검수 프로세스

```
Start
 └─▶ 바구니 스캔 (스캐너)
       └─▶ 바구니ID/주문 매핑 정보 조회 (PC)
             └─▶ 주문 내역 표시 (PC)
                   └─▶ RFID 리더 스캔 (PC)
                         ├─▶ 검수 미완료 ──▶ 반복
                         └─▶ 검수 완료 ──▶ 송장 출력
                                 └─▶ 적정 박스 선택 ──▶ 상품 포장 및 송장 부착
                                         └─▶ C/V 이적 ──▶ PLT 적재 ──▶ End
```

## 3. Wave 기반 작업 분배 개념

- WMS로부터 **Wave** 단위로 주문을 수신
- WCS가 Wave 내 주문을 분석하여 설비별 분류 기준에 따라 분배
- **대상 분류 기준**: 단포/합포/오더 피킹/수작업 등
- **DAS 최적화**: 유사 주문 분석 → 랙별 유사도 높은 주문으로 배치 → 분류 효율 극대화
  - 최적화된 주문과 그렇지 않은 주문은 동일 주문 수 대비 최대 **4배 이상** 성능 차이

## 4. Ex-PAS 반품 분류 프로세스

### 4-1. LV1 전체 흐름

```
RFID 반품 입고 검수
    └─▶ 반품 재고 확정 (WMS)
            └─▶ WAVE 생성 (WCS)
                    └─▶ 작업 게시 (WCS/관리/DAS)
                            ├─▶ 상품 투입 (제어)
                            └─▶ 중분류/소분류 (제어/DAS)
                                    └─▶ 포장/검수 (테이핑기/RFID)
```

### 4-2. 중분류/소분류 프로세스

```
Start
 └─▶ 상품 이동 및 슈트 분류 (PAS)
       └─▶ 상품 스캔 (DAS)
             └─▶ 표시기 Light On (DAS)
                   └─▶ 상품 분류 및 표시기 확정 (작업자)
                         ├─▶ 셀 미가득 ──▶ 상품 스캔 반복
                         └─▶ 셀 가득(Full) ──▶ 표시기 Fullbox
                                 └─▶ 반품 라벨 발행 및 부착
                                       └─▶ 내품 실적 전송 (DAS → WCS → RFID)
                                             └─▶ 박스 배출
```

## 5. DPS 주력 상품 버퍼 보관 프로세스

주력 상품을 분석하여 DPS 설비의 피킹 효율을 극대화하기 위한 재고 배치 프로세스입니다.

```
1단계: 상품별 출고일 계산 → 출고일 RANK (R1)
2단계: 상품별 주문건수 계산 → 주문건수 RANK (R2)
3단계: 최종 RANK = R1 × 출고일비율(60~70%) + R2 × 주문건수비율(30~40%)
4단계: 안전재고 = 상위 상품별 일평균 출고 수량 × 보관일
5단계: 현재 재고 < 안전재고인 경우 → 보충 수량 산출 → 보충 대상 목록 생성
```

> 분석 기간: 기본 90일 (시즌 변화 시 기간 축소 적용)
