#### 2.1.1 컴퓨팅 리소스 (EKS 기반 Kubernetes)

| 구성 요소 | 사양 | 수량 | 용도 |
|----------|------|------|------|
| **EKS Control Plane** | Managed | 1 | Kubernetes 마스터 |
| **Worker Nodes (AZ-A)** | c6i.4xlarge (16 vCPU, 32GB RAM) | 3대 | WCS 백엔드 Pod 실행 |
| **Worker Nodes (AZ-C)** | c6i.4xlarge (16 vCPU, 32GB RAM) | 3대 | HA 구성 |
| **Monitoring Node** | m6i.2xlarge (8 vCPU, 32GB RAM) | 2대 | Prometheus, Grafana |
| **Total vCPU** | - | 112 vCPU | - |
| **Total Memory** | - | 256GB | - |

**Pod 배포 전략**:
- WCS Backend Pod: 12개 (각 AZ에 6개)
- Pod 리소스: 2 vCPU, 4GB RAM per Pod
- HPA (Horizontal Pod Autoscaler): 12 → 24개 자동 스케일링
- PDB (Pod Disruption Budget): 최소 8개 항상 유지

#### 2.1.2 데이터베이스 (RDS PostgreSQL)

| 구성 요소 | 사양 | 수량 | 용도 |
|----------|------|------|------|
| **Primary DB** | db.r6g.4xlarge (16 vCPU, 128GB RAM) | 1대 | 마스터 DB (AZ-A) |
| **Standby Replica** | db.r6g.4xlarge (16 vCPU, 128GB RAM) | 1대 | 동기 복제 (AZ-C) |
| **Read Replica** | db.r6g.2xlarge (8 vCPU, 64GB RAM) | 2대 | 읽기 부하 분산 |
| **Storage** | io2 (Provisioned IOPS SSD) | 5TB | 30,000 IOPS |
| **Backup** | Automated Backup | 35일 | Point-in-Time Recovery |

**RDS 설정**:
- Multi-AZ 배포 (자동 Failover < 60초)
- Connection Pool: 500 connections
- 암호화: AES-256 (at rest), TLS 1.2 (in transit)

#### 2.1.3 캐시 (ElastiCache Redis)

| 구성 요소 | 사양 | 수량 | 용도 |
|----------|------|------|------|
| **Primary Cache** | cache.r6g.2xlarge (8 vCPU, 52GB RAM) | 1대 | 세션, 설정 캐시 (AZ-A) |
| **Replica** | cache.r6g.2xlarge (8 vCPU, 52GB RAM) | 1대 | 읽기 Replica (AZ-C) |
| **Cluster Mode** | Enabled | 3 shards | 분산 처리 |

**Redis 설정**:
- Cluster Mode 활성화 (3 shards × 2 replicas = 6 nodes)
- 자동 Failover 활성화
- 데이터 영속성: AOF + RDB 스냅샷

#### 2.1.4 메시지 브로커 (Amazon MQ - RabbitMQ)

| 구성 요소 | 사양 | 수량 | 용도 |
|----------|------|------|------|

| **Standby Broker** | mq.m5.2xlarge (8 vCPU, 32GB RAM) | 1대 | Active-Standby (AZ-C) |
| **Storage** | EBS (gp3) | 500GB | 메시지 영속성 |

**RabbitMQ 설정**:
- Active-Standby Failover (자동)
- 미러링 큐 활성화
- 메시지 처리량: 10,000 msg/s

## 5. 권장 구성안

### 5.1 쿠팡/대한통운급 대규모 물류센터 권장 방안

#### **✅ 권장: 클라우드 (AWS) 우선 채택**

**이유**:
1. **초기 투자 부담 없음**: $2.2M vs $0

6. **보안 컴플라이언스**: AWS ISO 27001, SOC 2 인증 활용

- 쿠팡, 대한통운 신규 풀필먼트 센터

  - 비용: $5K
