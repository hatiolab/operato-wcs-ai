#### 2.1.1 컴퓨팅 리소스 (EC2)

| 구성 요소 | 사양 | 수량 | 용도 |
|----------|------|------|------|
| **WCS Backend (Primary)** | m6i.large (2 vCPU, 8GB RAM) | 1대 | 메인 애플리케이션 서버 (AZ-A) |
| **WCS Backend (Standby)** | t3.large (2 vCPU, 8GB RAM) | 1대 | Hot Standby (AZ-C, 옵션) |
| **RabbitMQ Server** | t3.small (2 vCPU, 2GB RAM) | 1대 | 메시지 브로커 |
| **Total vCPU** | - | 6 vCPU | - |
| **Total Memory** | - | 18GB | - |

**인스턴스 설정**:
- Auto Scaling: 1 → 2개 (피크 타임 자동 확장)
- EBS Storage: gp3 100GB (OS + 앱)
- AMI: Amazon Linux 2023
- Security Group: 22 (SSH), 8080 (앱), 5672 (RabbitMQ)

---

#### 2.1.2 데이터베이스 (RDS PostgreSQL)

| 구성 요소 | 사양 | 수량 | 용도 |
|----------|------|------|------|
| **Primary DB** | db.t3.medium (2 vCPU, 4GB RAM) | 1대 | 마스터 DB |
| **Standby Replica** | db.t3.medium (Multi-AZ) | 1대 | 자동 Failover (옵션) |
| **Storage** | gp3 (General Purpose SSD) | 500GB | 20,000 IOPS |
| **Backup** | Automated Backup | 7일 | Point-in-Time Recovery |

**RDS 설정**:
- Single-AZ (기본) 또는 Multi-AZ (권장, +100% 비용)
- Connection Pool: 100 connections
- 암호화: AES-256 (at rest)

---

#### 2.1.3 캐시 (ElastiCache Redis)

| 구성 요소 | 사양 | 수량 | 용도 |
|----------|------|------|------|
| **Redis Cache** | cache.t3.small (2 vCPU, 1.5GB RAM) | 1 node | 세션, 설정 캐시 |

**Redis 설정**:
- Single Node (기본) 또는 Cluster Mode (3 shards, 옵션)
- 데이터 영속성: RDB 스냅샷 (일 1회)

---

#### 2.1.4 메시지 브로커 (RabbitMQ on EC2)

| 구성 요소 | 사양 | 수량 | 용도 |
|----------|------|------|------|
| **RabbitMQ Server** | t3.small (2 vCPU, 2GB RAM) | 1대 | 설비 제어 메시지 |
| **Storage** | gp3 50GB | - | 메시지 영속성 |

**RabbitMQ 설정**:
- EC2에 직접 설치 (Amazon MQ는 비용 대비 과도)
- Single Node (기본) 또는 3 Node Cluster (HA 필요 시)
- 예상 처리량: 100 msg/s

**비고**: Amazon MQ (관리형 RabbitMQ)는 최소 mq.t3.micro ($60/월)부터 시작하지만, 소규모에서는 EC2 직접 설치가 더 경제적

## 6. 실전 구성 예시

### 6.1 AWS 최소 구성 (월 $375)

```bash
# VPC 생성
aws ec2 create-vpc --cidr-block 10.0.0.0/16

# EC2 인스턴스 생성 (WCS Backend)
aws ec2 run-instances \
  --image-id ami-0c9c942bd7bf113a2 \
  --instance-type m6i.large \
  --key-name wcs-keypair \
  --security-group-ids sg-xxxxx \
  --subnet-id subnet-xxxxx \
  --block-device-mappings '[{"DeviceName":"/dev/sda1","Ebs":{"VolumeSize":100,"VolumeType":"gp3"}}]' \
  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=WCS-Backend}]'

# RDS PostgreSQL 생성
aws rds create-db-instance \
  --db-instance-identifier wcs-postgres \
  --db-instance-class db.t3.medium \
  --engine postgres \
  --master-username admin \
  --master-user-password <password> \
  --allocated-storage 500 \
  --storage-type gp3 \
  --backup-retention-period 7

# ElastiCache Redis 생성
aws elasticache create-cache-cluster \
  --cache-cluster-id wcs-redis \
  --cache-node-type cache.t3.small \
  --engine redis \
  --num-cache-nodes 1
```

### 6.2 온프레미스 구성 스크립트 (Ubuntu 22.04)

```bash
#!/bin/bash
# WCS 온프레미스 서버 구성 스크립트

# 시스템 업데이트
sudo apt update && sudo apt upgrade -y

# Java 17 설치
sudo apt install -y openjdk-17-jdk

# PostgreSQL 16 설치
sudo sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'
wget -qO- https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo tee /etc/apt/trusted.gpg.d/pgdg.asc &>/dev/null
sudo apt update
sudo apt install -y postgresql-16

# Redis 7 설치
sudo apt install -y redis-server
sudo systemctl enable redis-server

# RabbitMQ 설치
sudo apt install -y rabbitmq-server
sudo systemctl enable rabbitmq-server
sudo rabbitmq-plugins enable rabbitmq_management

# HAProxy 설치 (Load Balancer)
sudo apt install -y haproxy
sudo systemctl enable haproxy

# Prometheus + Grafana 설치
sudo apt install -y prometheus grafana
sudo systemctl enable prometheus grafana-server

# 방화벽 설정
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS
sudo ufw allow 9500/tcp  # WCS Backend
sudo ufw enable

echo "WCS 온프레미스 서버 구성 완료!"
```
